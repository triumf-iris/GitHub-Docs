# Contributing

These are the current best practices for contributing to the IRIS analysis codes. This document will evolve over time to adapt to the needs of the group.

## Best Practices
The main branch of each repository contains the most up-to-date versions of the analysis code. Any bug fixes or new features should be made to a separate branch or fork to maintain the integrity of the main branch. The changes can be merged into the main branch after testing and review.

1. Create a [new branch](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-and-deleting-branches-within-your-repository#creating-a-branch) or fork.
2. Use detailed commit messages to track your changes in the new branch or fork.
3. When you are confident that the bug is fixed or the feature is complete, [create a pull request](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request) with the main branch (base) and your new branch/fork (head). Use a descriptive title and description.
4. At least one other graduate student or postdoc must [review and approve](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/reviewing-changes-in-pull-requests/reviewing-proposed-changes-in-a-pull-request) your pull request. You and the reviewer(s) should work together to resolve any problems.
5. After approval, [merge the pull request](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/incorporating-changes-from-a-pull-request/merging-a-pull-request#merging-a-pull-request) to copy the new feature to the main branch (base).
6. [Delete](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-and-deleting-branches-within-your-repository#deleting-a-branch) the feature branch/fork (head).

## Permissions
The current oganization owners (as of March 2026) are Martin, Ritu, and Keilah. Only owners can [add/remove members](https://docs.github.com/en/organizations/managing-membership-in-your-organization/inviting-users-to-join-your-organization#inviting-a-user-to-join-your-organization) and modify organization settings.

All members have admin permissions (as of March 2026), which includes read/write access, creating branches and forks, and creating/approving/merging pull requests.
